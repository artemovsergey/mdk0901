# Теоретическая часть (краткое изложение):

1. **Корутины** — это легковесные потоки, которые позволяют писать асинхронный код в последовательном стиле.
2. **Структурированный параллелизм** обеспечивает отмену и распространение исключений через иерархию корутин.
3. **Функции-строители корутин**:
   - `launch` — запускает корутину без возврата результата.
   - `async` — возвращает `Deferred<T>`, который можно ожидать с помощью `await()`.
4. **Диспетчеры** определяют поток выполнения:
   - `Dispatchers.Main` — для работы с UI.
   - `Dispatchers.IO` — для сетевых и файловых операций.
   - `Dispatchers.Default` — для CPU-интенсивных задач.
5. **Приостанавливаемые функции** (`suspend`) могут быть остановлены и возобновлены без блокировки потока.

# Практический пример

```kotlin
import kotlinx.coroutines.*

fun main() = runBlocking {
    println("Start: ${Thread.currentThread().name}")

    val job = launch {
        withContext(Dispatchers.IO) {
            delay(1000)
            println("IO Work: ${Thread.currentThread().name}")
        }
        println("After IO: ${Thread.currentThread().name}")
    }

    job.join()
    println("End: ${Thread.currentThread().name}")
}
```

**Вывод:**
```
Start: main @coroutine#1
IO Work: DefaultDispatcher-worker-1 @coroutine#1
After IO: main @coroutine#1
End: main @coroutine#1
```

#### **Пример задания (вариант 1):**

**Задание:**  
Написать программу, которая выводит "Start", затем запускает корутину с задержкой 2 сек, выводит "Coroutine finished", и затем "End".

**Код:**
```kotlin
import kotlinx.coroutines.*

fun main() = runBlocking {
    println("Start")
    launch {
        delay(2000)
        println("Coroutine finished")
    }.join()
    println("End")
}
```

**Вывод:**
```
Start
Coroutine finished
End
```
